## Review Points

### Point 1: Mnemonic exposed in Docker Compose environment variables
The mnemonic is hardcoded in the compose file as an environment variable. While this is a CI test compose, the mnemonic is visible in plaintext. Add warnings that this mnemonic should never be used on mainnet.

- Add a ⚠️ WARNING in `docker/prism-test/README.md` in the Wallet section (Pre-configured Entities) stating the mnemonic is test-only and must never be used on mainnet
- Add a `# WARNING: Test-only mnemonic. Never use on mainnet.` comment in `tools/compose_gen/stacks/prism_test.py` next to the `wallet_mnemonic` variable
- Note: The compose YAML files are auto-generated (`# Code generated by Python script. DO NOT EDIT.`), so do NOT add comments there — the README and Python source are the source-of-truth locations

### Point 7: Semaphore serializes all submissions — document the design choice
`Semaphore::new(1)` in `lib/did-prism-submitter/src/dlt/embedded_wallet.rs` means only one transaction can be built/submitted at a time. This is intentional due to UTXO contention. Add a code comment explaining this design choice so future readers understand why concurrency is limited to 1.

### Point 8: Docs say "in-process" but embedded wallet runs as a subprocess
`docs/src/configuration/submitter.md` says the embedded wallet "sign and submit transactions in-process" but it actually spawns a Bun-compiled TypeScript binary as a subprocess. Fix the wording to say "as a subprocess" or equivalent, to be accurate about the architecture.
